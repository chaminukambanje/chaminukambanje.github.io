Module 2: Configure Financials

Chart of Accounts, Dimensions, and Posting Groups.

2.1 The Chart of Accounts (COA)

The backbone of the system. In BC, the COA is a flat list, but it uses indentation for structure.

  • Account Types:
    • Posting: Can accept transactions.
    • Heading: Visual separator only.
    • Begin-Total and End-Total: Used to sum up the accounts between them.
  • Direct Posting: Checkbox on the G/L Card. If unchecked, users cannot post directly to this account via Journal (it is locked for automated sub-ledger posting only).

2.2 Dimensions

Dimensions replace the need for a segmented chart of accounts.

  • Global Dimensions (2 Max): Flow to all ledgers. Can be used as a filter on G/L Entries. Available everywhere.
  • Shortcut Dimensions (6 Max): Available on document lines and lists, but less "indexed" than Global dimensions.
  • Dimension Priorities: If a Customer has a default dimension "Area: North" and the Item has "Area: South", you can set rules on which one wins or if it causes a conflict.

2.3 Posting Groups (The Matrix)

This maps "Who" and "What" to a G/L Account.

The General Posting Setup

This is a matrix table connecting:

  • Gen. Bus. Posting Group (Who): Assigned to Customer/Vendor.
  • Gen. Prod. Posting Group (What): Assigned to Items/Resources.

Result: When these two meet on an Invoice, the system looks at the matrix to find the Sales Account or COGS Account.

Exam Scenario: "You post a sales invoice but the Sales Account is missing."
Solution: Check the General Posting Setup matrix for the combination of the Customer's Business Group and the Item's Product Group.
