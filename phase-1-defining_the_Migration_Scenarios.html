Before touching any server, we must classify the project to select the correct tooling.

The Three Main Paths

Scenario Description Key Tooling
A. On-Premises to Cloud (SaaS) Moving from a local server (SQL) to Microsoft's managed cloud. Cloud Migration Tool
B. On-Premises Upgrade Moving from an old version (e.g., NAV 2017) to a newer On-Prem version. Application Merge Utilities (PowerShell) & Data Upgrade Codeunits
C. Tenant-to-Tenant Moving data from one SaaS environment to another. Environment Copy or Configuration Packages

Workflow: The Technical Assessment

Regardless of the path, you must perform a "Discovery" workflow to validate feasibility.

  1. Check Source Version Compatibility
    • Task: Verify the exact build number (e.g., NAV 2018, BC v14).
    • Rule: Direct migration requires Business Central v14 or later.
    • Action: If older, perform a "Technical Upgrade" first.
  2. Database Size Analysis
    • Task: Run SQL query to check table sizes.
    • Why: Storage limits apply (check for tables > 80GB).
    • Action: Truncate logs or attachments if necessary.
  3. Code Customisation Audit
    • Task: Identify C/AL vs. AL code.
    • Action: Export objects, compare against base app, and estimate rewrite time for Extensions.
