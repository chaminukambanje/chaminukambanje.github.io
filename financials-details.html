1. Chart of Accounts (COA)

The COA is the core list of all G/L accounts. In BC, this is a flat list, structured by indentation.

Account Categories & Subcategories

Used for reporting (Financial Reports). For example, you can map 5 different G/L accounts to the category "Cash". When you run the Balance Sheet, it sums them up automatically.

Direct Posting

This checkbox on the G/L Account Card is vital.

  • ON: Users can select this account in a General Journal (e.g., Office Supplies).
  • OFF: Users are blocked from selecting this account. It can only be posted to via the sub-ledgers (e.g., Receivables, Payables, Inventory). This prevents out-of-sync ledgers.

2. Posting Groups (The Matrix)

You must memorize the three main setup matrices.

A. General Posting Setup

Combines Gen. Bus. Posting Group (Customer/Vendor) + Gen. Prod. Posting Group (Item/Resource).

Defines:

  • Sales Account
  • Cost of Goods Sold (COGS) Account
  • Purchase Account
  • Inventory Adjustment Account (Losses/Gains)

B. VAT (Tax) Posting Setup

Combines VAT Bus. Posting Group + VAT Prod. Posting Group.

Defines:

  • VAT % (e.g., 20%)
  • VAT Calculation Type (Normal, Reverse Charge, Sales Tax)
  • Sales VAT Account / Purchase VAT Account

C. Inventory Posting Setup

Combines Location Code (Warehouse) + Inventory Posting Group (Item Category).

Defines:

  • Inventory Account (The Balance Sheet asset account)
Exam Question: "A user posts a Sales Invoice for a Customer (Domestic) and an Item (Retail). The system posts to the wrong Sales GL Account. Where do you fix it?"
Answer: Go to the General Posting Setup page. Find the line where Gen. Bus. Group = DOMESTIC and Gen. Prod. Group = RETAIL. Change the "Sales Account" column.

3. Dimensions

Dimensions categorize data (Department, Project, Area). They replace the need for segmented accounts (e.g., 4000-01-02).

Dimension Priorities

If a Customer has a default dimension "Area: North" and the Item has "Area: South", what happens?

  • Value Posting - Code Mandatory: The user MUST select a dimension, but it doesn't matter which one.
  • Value Posting - Same Code: The document requires a specific code.
  • Default Dimension Priorities: A setup page where you define that "Customer" dimension wins over "Item" dimension in a conflict.
