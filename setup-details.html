1. Creating and Configuring Companies

A "Company" in Business Central is a legal entity or a data container. A single BC environment (Tenant) can hold multiple companies (e.g., "Cronus USA", "Cronus Canada").

Types of Setup

  • Production vs. Sandbox: You should always configure in a Sandbox environment first. You can copy a Production environment to a Sandbox for testing, but you cannot copy Sandbox to Production directly.
  • Copy Company: In the "Companies" page, you can use the "Copy" function to duplicate an existing company (including data) to test a new process without affecting live data.

Assisted Setup

The "Assisted Setup" page is a hub for wizards. Key wizards to know for the exam:

  • Set up Company: Configures name, logo, and basic address.
  • Set up Exchange Rates: Connects to a service (like FloatRates or Yahoo) to auto-update currency rates.
  • Set up Email: Connects BC to M365 Outlook. You can set specific scenarios (e.g., Sales Orders come from sales@cronus.com, POs come from purchasing@cronus.com).

2. RapidStart (Configuration Packages)

This is the most critical setup topic. It involves migrating legacy data into BC.

The Migration Process (Order Matters!)

  1. Master Data (Dependents): Payment Terms, Posting Groups, Currencies, Locations.
    (Why? Because you can't create a Customer without a Payment Term Code).
  2. Master Records: Chart of Accounts, Customers, Vendors, Items.
  3. Open Balances: General Journal Lines (for G/L history), Item Journal Lines (for Inventory count).

Configuration Package Card - Key Fields

  • Table ID: The integer ID of the table (e.g., Customer = 18, Item = 27).
  • Dimensions as Columns: If checked, your Excel export will have columns for "Global Dimension 1" and "2". If you need Shortcut Dimensions 3-8, you must add the "Dimensions" table (360) separately.
  • Delete Table Records Before Processing: Warning! If checked, applying the package wipes the existing data in that table. Useful for cleaning a test company, dangerous for live go-live.
Exam Scenario: Import Errors
You import a package and get the error: "The field Gen. Bus. Posting Group of table Customer contains a value (DOMESTIC) that cannot be found in the related table (Gen. Business Posting Group)."

Solution: You failed to import the "Gen. Business Posting Groups" table first. You must create the group "DOMESTIC" before you can assign it to a customer.

3. Security Management

Users and Licenses

Users are managed in the Microsoft 365 Admin Center, not directly in BC. You must click "Update Users from Microsoft 365" in BC to sync them.

  • Full User (Essential/Premium): Full read/write access.
  • Team Member: Read-only access to most things; limited write access (e.g., can approve workflows, edit time sheets).

Permission Sets

  • System Permissions: Created by Microsoft (e.g., `D365 BASIC`, `D365 SALES`). You cannot edit these.
  • User-Defined Permissions: You can copy a System set and modify it.
  • Record Permissions: A tool that "watches" what you do. If you start recording and create a Customer, BC builds a permission set containing "Insert" permission for Table 18.

The "Effective Permissions" Page

If a user complains they cannot see a page, go here. It lists every permission a user has and Source (which Permission Set granted it).

4. Job Queues (Automation)

Job Queues allow you to schedule tasks to run automatically (e.g., Post Recurring Journals every night, Sync Exchange Rates every morning).

Setup Steps

  1. Object Type to Run: Usually "Report" or "Codeunit".
  2. Object ID: The ID number of the report (e.g., "Suggest Vendor Payments").
  3. Recurrence: Set days of the week and start/end times.
  4. Parameter String: Used to pass specific filters to the report (rarely used, but examinable).
Troubleshooting: If a Job Queue fails, check the "Log Entries". If it says "User does not have permission," check the permissions of the User ID assigned to the Job Queue Entry, not your own ID.
