1. Creating and Configuring Companies

Before any transaction occurs, the environment must be initialized. This often involves creating a new company within an existing environment.

Assisted Setup Wizards

Business Central provides wizards to streamline setup. You must know:

  • Set up Company: Basic details (Name, Address, Logo).
  • Set up Email: Connects BC to Exchange/Outlook (using Graph API or SMTP).
  • Set up Approval Workflows: Basic templates for document approvals.

2. RapidStart Services (Configuration Packages)

This is the primary method for data migration. It allows you to extract, transform, and load data.

The Workflow

  1. Create Package: Select the table ID (e.g., Table 18 for Customers).
  2. Select Fields: You rarely need all fields. Uncheck non-essential fields to reduce errors.
  3. Export to Excel: Generates a template.
  4. Populate & Import: Upload the filled Excel file.
  5. Validate: Critical Step. Checks business logic (e.g., Does the "Payment Terms Code" actually exist?).
  6. Apply: Moves data from the temporary staging table to the real database table.
Exam Scenario: You get an error "Parent ID not found".
Reason: You are trying to import "Sales Lines" before "Sales Headers," or you are importing Customers with a "Customer Posting Group" that hasn't been created yet.
Fix: Import Master Data (Posting Groups, Payment Terms) before Transactional Data.

3. Security Management

Business Central uses a layered security model.

Permission Sets vs. User Groups

  • Permission Set: A collection of permissions for specific objects (Tables, Pages, Reports).
    Example: "D365 SALES DOC" gives read/write access to Sales Orders.
  • User Group: A collection of Permission Sets.
    Example: The "SALES AGENT" group might contain "D365 SALES DOC" + "D365 CUSTOMER EDIT".

Recording Permissions

If you don't know which tables are needed for a task, use the Record Permissions feature. Start recording, perform the task (e.g., Create a Vendor), stop recording, and BC generates the permission set for you.

4. Number Series

Every document (Invoice, Order, Customer Card) needs a unique ID.

  • Default Nos.: Automatically assigns the next number (e.g., S-ORD-1001).
  • Manual Nos.: Allows the user to type a custom ID (e.g., using a customer's external ID).
  • Relationships: Allows you to select from multiple numbering series for a single document type (e.g., distinct series for Domestic vs. International orders).
