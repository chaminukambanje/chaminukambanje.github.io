Module 1: Set Up Business Central

Configuration packages, Users, and Core Settings.

1.1 Create and Configure a New Company

When starting a new implementation, you must decide how to populate data.

Configuration Packages (RapidStart)

This is the primary tool for data migration.

  • Format: Exports/Imports data via Excel (.xlsx) or XML.
  • Validate vs. Apply:
    • Validate: Checks the data against BC logic (e.g., does the Vendor Posting Group exist?).
    • Apply: Actually writes the data to the database tables.
  • Fields: You can exclude fields from the package to prevent overwriting existing data.
Exam Tip: If you get an error saying "Parent ID does not exist" during import, it usually means you are importing a child record (like a Sales Line) before the header (Sales Header) or related master data is missing.

Assisted Setup Wizards

Know which wizards are available out of the box:

  • Company Setup: Basic address and logo.
  • Email Setup: Connecting Exchange/Outlook.
  • Sales Tax Setup: Configuring tax areas/groups.

1.2 Manage Security

Users vs. User Groups

Users: Must be created in Microsoft 365 Admin Center first, then imported into BC.

User Groups: Collections of permission sets. Assign a user to a group (e.g., "D365 SALES") to give them all related permissions at once.

Permission Sets

  • Type: System (cannot change) vs. User-Defined (can edit).
  • Recording Permissions: You can "Record" your actions in BC to automatically generate a permission set based on what you clicked.
